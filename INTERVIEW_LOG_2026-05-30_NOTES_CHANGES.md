## Согласованные решения по пунктам

1. Выход из приложения по back-свайпу
- Область: везде.
- Таймаут подтверждения: 3 секунды.
- Текст: "Еще раз, чтобы выйти".

2. Подсветка нового содержимого
- Логика снятия подсветки: по таймеру после просмотра.
- Гранулярность:
  - Текст: по словам, если изменено слово.
  - Чеклист: по блокам, если добавлен новый пункт.
- Per-user метка "прочитано": не требуется.

3. Архивирование: toast + переход
- После архивации показывать toast текущего контекста.
- Action: "Перейти".

4. Выход из архива
- UX: кнопка выхода из архива в шапке.
- Дополнительно: одноразовая автоподсказка при первом входе в архив.

5. Свайп карточек слишком чувствительный
- Пороги: одинаковые для архива и удаления.
- Поведение: ослабить чувствительность, не отключать полностью.

6. Скрытие navbar в редакторе
- Подтверждено: скрывать navbar на целевых экранах редактирования.
- Фолбэк-навигация: уже есть.

7. Мультиселект заметок
- Операции bulk: все, кроме pin.
- Нужны: "Выбрать все" и "Снять выделение".
- Ограничение: только в текущем контексте/фильтре.

8. Realtime обновление списка заметок
- Обновлять: при открытом экране.
- Для фильтров/поиска: показывать баннер об изменениях.

9. Push по чеклистам
- Отправлять push только при полном completion списка.
- Повторные push: с cooldown.

10. Confetti
- Триггер: при полном completion.
- Кому показывать: всем.
- Доп. ограничений по min-пунктам: нет.

11. Неактивные chip
- Фон: как у фона заметки.
- Hover/active: оставить текущими.

12. Редактирование пункта чеклиста
- Режим: inline по двойному тапу.
- Отдельная кнопка Edit: не нужна.

13. Full-screen изображений в редакторе
- Удаление в full-screen: оставить.
- Индикатор: нужен (N/M).

14. Разархивирование с выбором локации
- Предвыбор: исходный контекст.
- Отмена выбора: заметка остается в архиве без изменений.

15. Аватары в "Просматривает"
- Fallback: 1 символ.
- Порядок: редактор первым.

16. Realtime внутри заметки + pull-to-refresh
- При конфликте с локальным вводом: спрашивать пользователя.
- Баннер о новых изменениях: нужен.

17. Gradient skeletons
- Первая волна: экран(ы) заметок.
- Визуальный стиль: мягкий shimmer/gradient.

18. Мультиселект изображений
- Для заметок: предлагать выбор "со сжатием/без сжатия".
- Лимит: до 10 изображений за операцию.

19. Toast по приглашениям
- При получении приглашения: да.
- При accept/decline отправленного приглашения: да.

20. Push при исключении/удалении группы
- Отправлять push: да.
- Deep-link с объяснением причины: не нужен.

21. Drag&drop чеклист-пункта
- Иконку drag скрыть.
- Long-press на весь контейнер: 250мс.
- Во время inline-редактирования: drag отключать.

22. Баг с пропадающей кнопкой создания заметки
- Диагностика: выполняется разработчиком.
- Ожидаемое поведение: FAB всегда видна на корне "Заметки", кроме archive mode.

23. Откат текста при автообновлении
- Правило: не перезаписывать локальный ввод автоматически.
- Кнопка diff-preview перед применением: не нужна.

## Базовые acceptance criteria

- Нет случайного закрытия приложения при одиночном back-жесте.
- Локальный ввод не теряется при входящих обновлениях.
- Архивный режим понятен и обратим (явный выход, явный переход).
- Чеклист completion корректно управляет push и confetti.
- Все agreed toast/push сценарии срабатывают в ожидаемых событиях.
- Скелетоны в заметках заменяют спиннер на загрузочных состояниях.
