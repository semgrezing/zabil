## Раунд 1 — Базовый контекст

### 1. Пользователи
- **Сейчас**: семья — 3 человека, возраст до 30 лет
- **Дальше**: друзья
- _Вывод_: маленький приватный круг доверия. Можно опираться на знакомство пользователей друг с другом. Не нужны: поиск чужих, защита от спама, публичные профили, продвинутая модерация.

### 2. Основные сценарии
- Запись **идей** (свободный текст)
- **Чеклисты**
- _Вывод_: два жанра контента, но они могут жить **в одной заметке** (см. B1).

### 3. Частота использования
- **Ежедневно**
- _Вывод_: продукт-привычка. UX обязан быть быстрым (создание заметки в одно касание). Загрузка экранов критична.

### 4. Что работает сейчас
- Ничего — приложение **очень сырое**.

### 5. Боли текущего UX
- **Сильные лаги** на экранах регистрации и авторизации
- **Неправильное позиционирование** фоновых изображений (`image 17`) и SVG (`Depth`) — макет: [Figma 12-633](https://www.figma.com/design/FG0OQ2LcuJob7QCMbtAYVX/Untitled?node-id=12-633)
- **Нет overlay/blur эффекта** в приложении, хотя есть в макете

### 6. Личные раздражения автора (критические баги)
- **Сессия не сохраняется** — после перезапуска нужно входить заново
- **Нет единства стиля** — экраны auth отличаются от остального приложения
- **Сломан функционал групп**:
  - Сейчас нельзя создать группу
  - Раньше приглашённый не мог ни принять, ни отклонить инвайт

### 7. Ближайшая цель
Полная развёртка приложения (desktop + mobile) в Figma:
- **UX flow в FigJam**
- **Экраны в Figma Design** — code-to-Figma в [файл, node 28-35](https://www.figma.com/design/FG0OQ2LcuJob7QCMbtAYVX/Untitled?node-id=28-35)

### 8. Post-MVP приоритет
- **Push-уведомления** (события см. D1)

### 9. Дедлайны
- Нет

## Раунд 2 — Уточнения

### A. Аудитория и устройства
- **A1**. Сейчас 3 пользователя, возраст до 30
- **A2**. Используют **и Android, и Windows равномерно** → desktop и mobile одинаково важны, нельзя относиться к одной из платформ как ко второстепенной.

### B. Модель контента
- **B1**. Заметка может содержать **и текст, и чеклист одновременно** (а-ля Notion / Apple Notes)
- **B2**. Есть **личные заметки** (только для меня) и **групповые** (для всех в группе).
  - Создание идёт в **текущий выбранный контекст**: если на экране заметок выбрана группа — создаётся в группе; если выбрано «Личное» — в личных.
  - **Нужен функционал переноса заметок** между группами / личным.
- **B3**. Создание заметки — **в одно касание**. Контекст (группа/личное) определяется текущим выбором на экране заметок.

### C. Визуальный язык
- **C1**. Стиль экранов auth (тёмный, кинематографичный) — **новый общий язык всего приложения**. НО фоновое изображение остаётся **только на экранах auth**. Внутри приложения — тот же темный кинематографичный язык без фотофонов.
- **C2**. Референсы — собираем в отдельном файле [REFERENCES.md](REFERENCES.md), пользователь будет наполнять.

### D. Push-уведомления (события)
Все следующие:
- Меня пригласили в группу
- В группе появилась новая заметка
- Кто-то отметил пункт в общем чеклисте
- Кто-то изменил заметку, в которой я соавтор

### E. Последовательность работы (утверждена)
1. **Фикс критических багов** (персистентность сессии, лаги auth, группы, фон/blur)
2. **Единая дизайн-система** (токены, компоненты, тёмная/светлая темы) — на основе стиля auth-экранов
3. **UX flow в FigJam** (карта экранов и переходов, оба формата — mobile + desktop)
4. **Code-to-Figma** — перенос всех экранов в [Figma 28-35](https://www.figma.com/design/FG0OQ2LcuJob7QCMbtAYVX/Untitled?node-id=28-35)
5. **Push-уведомления**

## Раунд 3 — Финальные уточнения

### F. Первый запуск (онбординг)
- **Сразу текущий экран авторизации.** Никаких welcome-карусели или splash с CTA. Изменений в экран auth не вносим.

### G. «Личное» в UI
- **Псевдо-группа в том же селекторе**, рядом с реальными группами. Единообразный паттерн выбора контекста.

### H. Поиск
- **Глобальный с фильтрами по группам.** Один поиск находит везде; чипы-фильтры сверху позволяют сузить.

### I. Архив
- **Раздельный по контексту + общая вкладка «Весь архив».** По умолчанию архив текущей группы/личного, но есть переключатель «показать всё».

### Интервью по пунктам

#### 1) Перенос заметок между группами
- Статус: done
- Ответы:
  - Перенос нужен между любыми контекстами: группа -> группа, группа -> Личное, Личное -> группа.
  - В групповом контексте перенос может выполнять любой участник группы.
  - Данные при переносе сохраняются 1:1 (текст, чеклист, вложения, метаданные).
  - Точки входа из заметки: и в kebab-меню, и отдельной кнопкой в хэдере.

#### 2) Личное пространство заметок
- Статус: done
- Ответы:
  - Личное пространство полностью приватно: другим пользователям недоступны даже метаданные.
  - Вход в личные заметки нужен одновременно по 2 каналам: через чип «Личное» и через кнопку в хэдере.
  - Кнопка «Личное» в хэдере переключает на личный контекст и фиксирует фильтр.

#### 3) Последнее сообщение в Чаты > Группы
- Статус: done
- Ответы:
  - Формат превью: «Автор: текст + время».
  - Если сообщений нет: показывать CTA «Создать первое сообщение».

#### 4) Связка note-chat и group-chat
- Статус: done
- Ответы:
  - Сообщения note-chat дублируются в group-chat.
  - Сообщения из заметок должны иметь легкую визуальную маркировку с названием заметки и ее цветовым маркером (не тяжелый бейдж).
  - Из group-chat в note-chat: переход по сообщению/маркеру в связанный чат заметки.
  - Из note-chat в group-chat: кнопка-иконка чата в хэдере.

#### 5) Удаление вкладки «Группы"
- Статус: done
- Ответы:
  - Вкладка «Группы» удаляется из bottom-nav.
  - Переносится весь текущий функционал вкладки «Группы» в хэдер «Чаты» без потери возможностей.
  - В хэдере «Чаты» добавляется иконка «Группы», расположение: слева от кнопки добавления пользователей.

#### 6) Настройки группы
- Статус: done
- Ответы:
  - Настройки группы доступны владельцу и администраторам.
  - Ограничение исключения участников: нельзя исключать владельца и администраторов.

#### 7) Аватарки: fullscreen + история + удаление
- Статус: done
- Ответы:
  - Требование распространяется на все сущности, где есть аватарка.
  - Fullscreen режим должен включать просмотр, листание истории и удаление.
  - История аватарок без лимита.
  - Удаление текущей аватарки разрешено, после удаления ставится placeholder.

#### 8) Кадрирование аватарок
- Статус: done
- Ответы:
  - Формат кропа: круг 1:1 для всех аватарок.
  - В кроппере нужны все стандартные controls (масштаб, поворот, сброс, предпросмотр, выбор области, подтверждение/отмена).

#### 9) Chips в «Заметки"
- Статус: done
- Ответы:
  - Для активной chip в «Заметки» текст должен быть черным.
  - Checkmark выбранной chip убрать.
  - Правило применять консистентно во всех местах с этим паттерном.

#### 10) Изображения в чатах
- Статус: done
- Ответы:
  - Нужен множественный выбор изображений.
  - Нужны 2 режима отправки, как в Telegram: со сжатием и без сжатия.
  - Лимиты по количеству/размеру не ограничиваются продуктом («любое»).

#### 11) Push-уведомления
- Статус: done
- Ответы:
  - Push должны работать на Android и Windows.
  - Обязательные события: инвайт, новая заметка, изменения заметки/чеклиста, новые сообщения в личных/групповых/note-чатах.
  - При тапе по push открывать связанный экран события.
  - Нужны звук и badge-счетчик.
